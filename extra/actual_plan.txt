FASE 01: Setup, TypeScript & Database (Giorni 1–7)

- Giorno 1: Architettura & ambiente di sviluppo
  - Installare Node.js 20, VS Code, Git, Postman, MongoDB Compass, pgAdmin.
  - Creare il monorepo: SynapsisForge-hub/backend/, frontend/, infra/.
  - Disegnare su carta o Excalidraw l'architettura completa.
  - Inizializzare il repository Git, creare .gitignore per Node e il file .env.example.

- Giorno 2: Docker Compose — solo infrastruttura DB
  - Creare infra/docker-compose.yml con tre servizi: postgres:16, mongo:7, redis:7.
  - Configurare volumi per la persistenza dei dati, esporre le porte standard (5432, 27017, 6379).
  - Avviare con docker-compose up -d e verificare le connessioni.
  - Creare il database SynapsisForge in PostgreSQL e sperimentare i tipi UUID, JSONB, TIMESTAMPTZ.

- Giorno 3: TypeScript — tipi, interfacce, generics
  - Studiare tipizzazione statica, differenza tra interface e type, enum, generics di base.
  - Esercizio: riscrivere in TypeScript una classe JavaScript che già si conosce.
  - Capire perché TypeScript è obbligatorio in NestJS.

- Giorno 4: TypeScript — decoratori, utility types
  - Studiare class decorator e method decorator, Partial<T>, Pick<T>, Omit<T>, union types, type guards.
  - Esercizio pratico: scrivere un decoratore custom @Log che stampa in console gli argomenti di ogni metodo che decora.
  - Esplorare il file tsconfig.json: capire le opzioni strict, experimentalDecorators, emitDecoratorMetadata.

- Giorno 5: PostgreSQL vs MySQL & MongoDB vs SQL — confronto guidato
  - Studiare differenze: PostgreSQL ha JSONB con indici, UUID nativo; MySQL è più semplice ma meno espressivo.
  - Studiare differenze: MongoDB non ha schema fisso (ideale per contenuto lezioni), PostgreSQL per relazioni forti.
  - Eseguire query di esempio su entrambi i database per familiarizzare con le sintassi diverse.

- Giorno 6: Schema del database — progettazione
  - Disegnare il diagramma ER completo: users, courses, lessons, enrollments, reviews, certificates, payments, categories.
  - Decidere cosa va in PostgreSQL (relazioni, pagamenti, iscrizioni) e cosa in MongoDB (contenuto lezioni).
  - Disegnare anche la struttura del documento MongoDB per una lezione.

- Giorno 7: Schema DB & definizione API [MILESTONE]
  - Revisionare il diagramma ER e correggere eventuali relazioni mancanti o cicliche.
  - Elencare tutti gli endpoint REST che l'app dovrà esporre.
  - Creare il file API_SPEC.md nel repo con la lista degli endpoint.

FASE 03: Autenticazione — JWT, OAuth2 (Google/GitHub), RBAC (Giorni 25–35)

- Giorno 25: JWT — teoria e setup Passport.js
  - Studiare anatomia JWT, algoritmo HMAC-SHA256.
  - Capire dove salvare i token: access token in memoria JS, refresh token in cookie HttpOnly.
  - Installare: @nestjs/passport passport passport-jwt @nestjs/jwt bcrypt.
  - Creare AuthModule con JwtModule.registerAsync().

- Giorno 26: Implementazione login & JWT emissione
  - Endpoint POST /auth/login: verifica email + password con bcrypt, genera access e refresh token.
  - Creare JwtStrategy che estende PassportStrategy.
  - Creare JwtAuthGuard: applicare globalmente e usare decoratore @Public().
  - Salvare l'hash del refresh token sul DB User.

- Giorno 27: Refresh token & logout
  - Endpoint POST /auth/refresh: riceve il refresh token dal cookie, emette nuovo access token.
  - Endpoint POST /auth/logout: rimuove l'hash del refresh token dal DB.
  - Configurare la risposta del login per impostare il cookie HttpOnly.
  - Testare il ciclo completo con Postman.

- Giorno 28: OAuth2 — teoria flusso Authorization Code
  - Studiare i 4 grant types OAuth2.
  - Tracciare il flusso Google passo per passo.
  - Registrare le app su Google Cloud Console e GitHub Developer Settings.

- Giorno 29: OAuth2 — implementazione Google login
  - Installare passport-google-oauth20 e creare GoogleStrategy.
  - Nel metodo validate(), creare l'utente se non esiste.
  - Endpoint GET /auth/google e GET /auth/google/callback.
  - Testare il flusso completo aprendo il link nel browser.

- Giorno 30: OAuth2 — GitHub login
  - Installare passport-github2 e creare GithubStrategy.
  - Endpoint GET /auth/github e GET /auth/github/callback.
  - Gestire il caso in cui GitHub non fornisce email pubblica.
  - Testare entrambi i provider.

- Giorno 31: RBAC — decoratori custom e RolesGuard
  - Creare il decoratore @Roles() usando SetMetadata.
  - Creare RolesGuard: usa Reflector per leggere i ruoli dai metadati.
  - Configurare l'ordine corretto: prima JwtAuthGuard, poi RolesGuard.
  - Applicare @Roles("instructor") agli endpoint di creazione/modifica corsi.

- Giorno 32: Registrazione & email verification
  - Endpoint POST /auth/register: crea utente con password hashata.
  - Endpoint GET /auth/verify-email/:token: attiva l'account.
  - Endpoint POST /auth/password/reset e POST /auth/password/confirm.
  - Aggiungere campo isVerified su User.

- Giorno 33: CORS & security headers
  - Configurare app.enableCors() in main.ts.
  - Installare helmet e abilitarlo in main.ts.
  - Testare CORS con una chiamata fetch dal browser.

- Giorno 34: Test Auth — tutti gli scenari
